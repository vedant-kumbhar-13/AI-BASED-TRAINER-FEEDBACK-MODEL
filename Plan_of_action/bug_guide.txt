[ ]  Copy the file URL from the response and open it in a browser tab. The PDF downloads — no 404.

Resume Parsing (BUG-D + BUG-E)
[ ]  Upload a resume that lists Python, Django, React as skills. After parsing, the skills array in the API response contains "Python", "Django", "React" — not an empty array [].
[ ]  Check the skills array appears in the interview questions — at least one question should mention a technology from the resume.
[ ]  In the backend terminal, the debug log should NOT show "# Limit to prevent token overflow" appearing in the Gemini request prompt.

Voice Interview — Double-Advance (BUG-B)
[ ]  Start an interview. Answer Q1 by speaking. Stop speaking for 3 seconds.
[ ]  Within 1 second of the silence triggering, immediately click "Next Question".
[ ]  Result: you land on Q2. NOT Q3. Q1 answer is saved in collectedAnswers.
[ ]  Click "Review Answers" at the end. Q1 shows your actual spoken answer — not "[No answer provided]".

Authentication (BUG-F)
[ ]  Log in. Refresh the browser. You remain logged in — no redirect to /login.
[ ]  In the browser console, run: localStorage.getItem("access_token") — paste the value into jwt.io and confirm it is a valid token.
[ ]  No console errors mentioning "Invalid character" or "atob" during login or page load.
